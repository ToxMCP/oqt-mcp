# O-QT MCP v0.2.0

This is a maturity and contract release for O-QT MCP. It does not redesign the module's role. It makes the repo identity, published contracts, and downstream orchestration story match the architecture that already exists.

## Highlights

- Re-aligned package metadata, docs, and repository references around `ToxMCP/oqt-mcp`.
- Clarified the public surface model:
  - `run_oqt_multiagent_workflow` is the primary default entrypoint.
  - Lower-level Toolbox tools remain public as expert/debugging helpers.
  - Final suite-level synthesis stays outside O-QT MCP.
- Added portable handoff schemas for:
  - `oqtWorkflowRecord.v1`
  - `oqtHazardEvidenceSummary.v1`
  - `oqtReadAcrossSummary.v1`
- Added `portable_handoffs` directly to the high-level workflow responses.
- Added `build_portable_handoffs_from_log` so downstream systems can reconstruct schema-aligned handoffs from stored `log_json` without rerunning the Toolbox.
- Added architecture and downstream-orchestration docs to make the suite boundary explicit.

## Included Changes

### Documentation and identity

- README rewritten to match the ToxMCP module pattern more closely.
- Quick start now points to `https://github.com/ToxMCP/oqt-mcp`.
- Version bumped to `0.2.0`.
- Release process updated to treat schema validation as part of release hygiene.

### Contract layer

- Published portable schemas under `schemas/`.
- Added validated example instances for each published schema.
- Added regression tests to keep examples, schemas, and runtime handoffs aligned.

### Runtime behavior

- `run_oqt_multiagent_workflow` now returns:
  - live MCP artifacts
  - `portable_handoffs.oqtWorkflowRecord.v1`
  - `portable_handoffs.oqtHazardEvidenceSummary.v1`
- `build_grouping_justification` now returns:
  - live MCP artifacts
  - `portable_handoffs.oqtWorkflowRecord.v1`
  - `portable_handoffs.oqtReadAcrossSummary.v1`
- New helper tool:
  - `build_portable_handoffs_from_log`

## Boundary reminder

O-QT MCP remains the ToxMCP suite's specialized OECD QSAR Toolbox engine.

It does not become:

- the suite orchestrator
- the final BER/WoE synthesis layer
- the home for PBPK, AOP, or CompTox logic
- an async queue/persistence platform in this release

That work belongs above O-QT in a downstream orchestrator.
